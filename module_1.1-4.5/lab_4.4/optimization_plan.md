# Карта проблем (Issues Map) — аудит производительности

## Как проводился аудит

- **Lighthouse** (Chrome DevTools → вкладка «Производительность» / Lighthouse): ориентиры по **LCP**, **INP**, **CLS**, общий **Performance score**, список аудитов и рекомендаций.
- **Network** с throttling **Slow 4G / Fast 3G**: перезагрузка страницы, разбор **waterfall** — что блокирует первый кадр, что скачивается дольше всего, есть ли **render-blocking** CSS/JS.
- **Performance** (запись профиля): поиск **Long Tasks** (>50 ms) на главном потоке при загрузке и после интеракций.
- **Bundle Analyzer** (`rollup-plugin-visualizer` / аналог): какие **npm-пакеты** и **собственные модули** дают наибольший вклад в размер бандла.

Ниже — не менее пяти конкретных находок, сопоставленных с метриками и приоритетом устранения.

## Таблица: проблема → метрика → приоритет

| Проблема | Метрика, на которую влияет | Приоритет |
|----------|----------------------------|-----------|
| Главный чанк `main`/vendor **~1.0–1.2 МБ** (фреймворк + UI-kit + утилиты в initial bundle) | **LCP** (откладывает отрисовку и гидрацию), **INP** (долгий парсинг/компиляция/исполнение на main thread) | **Высокий** |
| **Render-blocking**: крупный CSS в `<head>` без inline critical CSS; синхронные шрифты без `font-display` | **LCP** | **Высокий** |
| Hero/LCP-изображение: **без фиксированных размеров или без современного формата**, нет `preload` для LCP-ресурса | **LCP**, **CLS** | **Высокий** |
| Тяжёлая страница (например, **графики**): библиотека попадает в initial graph вместо **lazy route / dynamic import** | **INP**, **LCP** | **Высокий** |
| **Long tasks** при инициализации чарта и нормализации данных (всё на main thread) | **INP** | **Средний** |
| Сторонние скрипты (аналитика, чат-виджет) — **блокирующие или без `async`/`defer`**, лишние DNS/TLS | **LCP**, **INP** | **Средний** |
| Дублирующиеся зависимости / **неполное tree-shaking** (импорт «целого пакета» вместо точечного entry) | **LCP**, **INP** | **Средний** |

*Примечание:* точные числа размеров и миллисекунд подставьте из своих замеров Lighthouse и отчёта бандл-анализатора на **вашем** задеплоенном приложении.
