1. Оптимизация приложения

ИИ подсказал несколько улучшений:

разделить код через lazy loading (Vue Router уже использует динамические импорты)
добавить lazy loading для изображений (loading="lazy")
вынести тяжелые компоненты в отдельные чанки
добавить базовые meta-теги для SEO (title, description, viewport)
улучшить производительность через минимизацию повторных ререндеров
2. Деплой

Для деплоя я использовал Netlify, так как он проще для SPA на Vue.

Главное — настроить редиректы, чтобы Vue Router (history mode) работал корректно.

3. Архитектурный вывод

Использование готовых компонентов сильно ускорило разработку SPA.

Плюсы:

быстрое переиспользование UI
единый стиль интерфейса
меньше дублирования кода

Минусы:

иногда сложно адаптировать старые компоненты под новую архитектуру
появляется зависимость от правильной структуры UI-кита
4. Эволюция промптов

Промпты стали намного сложнее и точнее.

Если раньше я просто просил “сгенерируй компонент”, то теперь я описываю:

структуру приложения
состояние
поведение
интеграцию с store и роутером

Самый удачный пример — промпт для TasksPage, где я сразу описал:

Pinia store
структуру UI
действия (add/remove/toggle)
использование AppButton
5. Роль разработчика в AI-first SPA
Архитектор структуры приложения (связи между страницами, store, компонентами)
Контролёр качества кода (проверка логики, edge cases, безопасность)
Дизайнер поведения системы (что именно должен делать UI, а не как писать код)
