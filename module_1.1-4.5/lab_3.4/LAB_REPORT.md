# LAB_REPORT.md — lab_3.4

## 1. Какая фича была добавлена
Я добавил кэширование ответа API в `localStorage` на 5 минут.  
Это уменьшает количество запросов и ускоряет загрузку приложения при повторном открытии страницы.

Кэш работает так:
- при первом запросе данные сохраняются вместе с timestamp
- при следующих заходах проверяется срок хранения (5 минут)
- если кэш актуален — данные берутся из localStorage
- если истёк срок — делается новый запрос к API

Также добавил кнопку "Обновить данные", которая принудительно обновляет данные и не использует кэш.

---

## 2. Рефакторинг кода
После ревью от ИИ я вынес всю логику загрузки данных и состояния в отдельный composable `useWeatherApi()`.

Теперь:
- `App.vue` отвечает только за UI и отображение состояния
- `useWeatherApi.js` содержит:
  - fetch-запрос
  - обработку ошибок
  - isLoading/error
  - кэширование
  - функцию принудительного обновления

Код стал чище и проще расширять.

---

## 3. Самые сложные моменты при работе с API
Основные сложности были:
- продумать обработку ошибок сети (offline режим)
- правильно форматировать данные daily (там массивы, и нужно связывать индексы)
- сделать UI понятным: чтобы пользователь видел, что происходит (loading/error)

---

## 4. Как изменился подход к промптам
В начале я писал промпты максимально просто (типа "сделай запрос и выведи данные").  
Но по ходу работы понял, что ИИ лучше работает, если сразу задавать детали.

Сейчас я сразу добавляю в промпт:
- структуру state (isLoading/error/data)
- try/catch и обработку response.ok
- условный рендеринг
- требования к UX (Loader, ErrorAlert, retry button)
- оптимизации (кэширование, не пересоздавать компоненты без нужды)

То есть промпты стали более похожи на нормальное техзадание.

---

## 5. Вывод о роли разработчика в AI-first процессе
После лабораторной я понял, что ИИ реально ускоряет работу, но без контроля он легко генерирует код "в лоб".

Главные обязанности разработчика в AI-first процессе:
1) Проектировать архитектуру и структуру компонентов заранее.
2) Проверять логику работы с данными и edge cases (ошибки сети, пустые ответы).
3) Делать финальную доработку: оптимизация, читаемость, UX и поддерживаемость.

ИИ — это хороший исполнитель, но ответственность за итоговое качество всё равно на разработчике.
