# Итоговый отчёт — транспортный дашборд (модуль 5.2)

## Выбор технологий

- **Карта:** Leaflet + OpenStreetMap тайлы — быстрый старт и предсказуемое API; `leaflet.markercluster` для агрегации при отдалении и 200+ маркеров. Альтернатива с высокой производительностью — MapLibre GL, здесь не использована ради простоты учебного контура.
- **Графики:** Chart.js через `vue-chartjs` — компактные обновления при смене диапазона (`animation.duration` ограничен).
- **Чат-агент:** детерминированный разбор намерений и сущностей в `transportAnalystAgent.js` без внешнего LLM: проще сдавать как воспроизводимый MVP; слой LLM можно подключить только на этапе классификации намерения.

## Архитектура агента и взаимодействие

1. Пользователь вводит текст в `ChatWidget.vue`.
2. `processTransportQuery` классифицирует намерение, извлекает маршрут/тип ТС/регион/порог минут.
3. Формируется массив команд (`UPDATE_MAP_FILTER`, `FETCH_AND_RENDER_CHART`, `SET_DELAY_THRESHOLD`, …).
4. `dashboardStore.applyCommand` и `uiStore` меняют состояние.
5. `TransportMap`, `RouteLoadChart`, `StopsCompareChart` реагируют на стор и данные API.
6. В чат уходит текстовый отчёт (количества, подтверждение действия).

Схемы — в `ARCHITECTURE.md`.

## Нагрузочное тестирование (локально)

| Сценарий | Наблюдение |
|----------|------------|
| 88 ТС по умолчанию | Карта плавная, CPU умеренный |
| `GET /api/vehicles?count=250` | Маркеры кластеризуются при отдалении; без кластера возможны лаги на слабых машинах |
| Смена диапазона истории на графике | Перерисовка Chart.js за счёт ограниченной анимации |

Вывод: основной риск — частые полные перерисовки Vue при большом числе узлов DOM; снижается императивным обновлением маркеров Leaflet и отделением NLP от потока координат.

## Самоанализ (вопросы блока 4)

**Граница ответственности бота и диспетчера.**  
Бот подсказывает и настраивает визуализацию на основе данных и правил; **решение по операциям на линии** (сдвиги, связь с пассажирами, обход инцидентов) остаётся за диспетчером / руководителем смены. Ошибочная интерпретация данных — зона ответственности организации: должны быть политики валидации, логирование команд и источников.

**Новые специализации.**  
«AI-оркестратор диспетчерских систем», «инженер по человеко-ориентированной аналитике транспорта», «архитектор потоковых данных + LLM guardrails».

**Развитие в мультиагентную систему.**  
Выделить: **агент-картограф** (геофильтры, слои), **агент временных рядов** (прогноз загрузки), **агент отчётов** (PDF/CSV), координируемых **главным диспетчером** с политикой конфликтов и приоритетом безопасности.
