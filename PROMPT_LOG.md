# Промпт-дневник (этап 2) — 3 самых сложных промпта

## Промпт 1 — `agentCore.js` (намерения + слоты)

**Текст (суть):**  
«Реализуй `analyzeIntent` для русского языка: интенты find/book/cancel/where/cheapest; извлекай номер входа (цифра или слово), длительность в часах, час начала "с 15:00", ключевые слова "дешевле/дешёвле". Верни строго типизированную структуру. Без LLM, только regex и словари синонимов. Покрой граничные случаи: пустая строка, несколько чисел.»

**Почему так:**  
Нужен **воспроизводимый** контракт для оркестратора и тестов; смешение языка и логики легко даёт дрейф поведения, поэтому в промпте явно зафиксированы интенты и формат ответа.

**Итерации:**  
Сначала модель распознавала только «вход N» латиницей — добавили **русские формы** («у третьего входа»). Второй проход: уточнили приоритет **book vs find**, если в одной фразе есть и «найди», и «забронируй».

## Промпт 2 — `ParkingMap.vue` + `v-memo`

**Текст (суть):**  
«Сетка мест 8×12 из Pinia, цвета статусов, `highlightedSpotId` с обводкой. Ячейка — отдельный дочерний компонент с `v-memo="[spot.status, spot.price, isHighlighted]"`. Обновление каждые 10 сек не должно перерисовывать всю сетку без нужды.»

**Почему так:**  
Задание требует **оптимизацию рендера** при частых изменениях датчиков; `v-memo` привязан к наблюдаемым полям, чтобы не оптимизировать «вслепую».

**Итерации:**  
Первый вариант мемоизировал весь ряд — при смене одной ячейки ломалась подсветка соседей. Перешли на **мемо на ячейку** + стабильный `key` по `spot.id`.

## Промпт 3 — Playwright E2E «чат → карта → бронь»

**Текст (суть):**  
«Напиши Playwright-тест: `goto('/')`, `getByTestId('chat-input').fill(...)`, отправка, `expect` сообщения ассистента, `getByRole('button', { name: 'Забронировать' }).click()`, проверка `booking-card` и текста подтверждения. Учти асинхронность ответа (ожидание сетевого/таймаута не требуется — всё локально).»

**Почему так:**  
Сценарий — сквозной UX; в промпте перечислены **селекторы по ролям и testid**, чтобы тесты не ломались от косметики CSS.

**Итерации:**  
Первый прогон падал из-за **двойного** сообщения пользователя — добавили `press('Enter')` вместо повторного `click`. Второй — флейки из-за отсутствия `await expect(...).toBeVisible()` для кнопки брони.
